# Simulink + ROS 长期学习规划（研一暑假 → 研二秋招）

> 规划时间：2026年7月 — 2027年9月（约14个月，62周）
> 每日投入：30~45分钟（可灵活调整）
> 总可用时间：约 300~400 小时
> 核心目标：以 Simulink 代码生成能力为主战场，ROS 集成为副线，找到机械/控制/机器人方向好工作

---

## 设计理念

**不是"什么都学一点"，而是"在少数高价值技能上做到比90%的求职者深"。**

```
市场现实：
  会 Simulink 基础操作的人    → 很多，不值钱
  会 Simulink + 代码生成的人  → 很少，值钱
  会 Simulink + ROS 集成的人  → 更少，更值钱
  三者全通 + 有实际项目       → 稀缺，面试强竞争力
```

**时间分配权重**：基础阶段快速通过 → 代码生成全力投入 → ROS 集成打通工作流 → 项目实战巩固

## 贯穿全程的习惯（从第一天开始）

### 1. Git/GitHub 习惯
每个练习模型、每段生成的代码、每个项目的 README 都提交到 Git。
- 面试时直接给链接，比说"我会"有力100倍
- 从 Phase 1 第一天就开始，养成肌肉记忆

### 2. 面试素材积累
从 Phase 3 开始，每周花15分钟回答自己一个问题并记录：
- "这周学的东西，面试官问我我能讲清楚吗？"
- 答不出来的就是还没真正理解，回去补

### 3. ROS 环境提前搭建
在 Phase 1-2 期间（不占学习时间），把 Ubuntu 22.04 + ROS2 Humble 装好（双系统或虚拟机）。
- 等到 Phase 5 要用时，环境已经就绪
- 安装本身不花学习时间，只花下载和配置时间

### 4. 简历迭代
每阶段结束时更新一版简历，不要等到最后才写：
- Phase 1 结束：写第一版（基础技能）
- Phase 3 结束：加代码生成技能
- Phase 5 结束：加 ROS 集成技能
- Phase 6 结束：加项目经历
- Phase 7：精修终版

## Phase 1：Simulink 基础建模（第1~5周，2026年7月）

### 目标
能独立搭建闭环控制系统模型。基础够用就行，不追求完美——后面用到了自然会熟。

### 周计划

| 周次 | 主题 | 每天做什么 | 周末验证 |
|:---:|:----|:---------|:--------|
| 1 | 入门 + 基本操作 | 2天 Simulink Onramp（免费官方课程）；2天练 Sine Wave→Scope、Constant/Gain/Add；1天看 Getting Started 视频 | 搭出正弦波示波器模型 |
| 2 | 传递函数 + 状态空间 | 3天搭 1/(s+1)、1/(s²+2s+1) 看阶跃响应，对比过阻尼/临界/欠阻尼；2天搭弹簧-质量-阻尼 State-Space 模型 | 截图三种阻尼响应对比 |
| 3 | PID Controller | 2天理解 P/I/D 作用；2天搭 PID + 二阶被控对象；1天手动调参 | PID 调参后响应收敛 |
| 4 | **闭环位置跟踪 + 子系统** | 3天搭完整闭环：目标→PID→Plant→反馈→Scope；2天封装子系统 + Mask | 层级化模型带参数面板 |
| 5 | MATLAB Function + **阶段考核** | 2天写自定义控制律函数；3天从零搭一个新场景（倒立摆/水箱/RLC电路） | 独立完成一个全新模型 |

### 简历产出
```
熟练使用 Simulink 进行动态系统建模与仿真
掌握 PID 控制系统设计与参数整定方法
具备模型层级化设计与子系统封装能力
```

### 资源
- Simulink Onramp（前2天用完就不再需要）
- 官方文档（遇到问题直接查）
- 不需要买书

## Phase 5：ROS + Simulink ROS Toolbox（第26~31周，2027年1月~2月）

### 目标
打通 Simulink → ROS 节点 → 机器人部署的完整工作流。
不是从零学 ROS，而是用你已经有的 Simulink + 代码生成能力，通过 ROS Toolbox 直接切入 ROS 生态。

### 为什么这个阶段值钱

MathWorks 的 **ROS Toolbox** 可以直接把 Simulink 模型生成 **ROS 节点（C++）**，部署到真实机器人。
你的角色不是"手写 ROS 节点"，而是"在 Simulink 里设计控制算法 → 一键生成 ROS 节点 → 部署到机器人"——这叫 **Model-Based Design with ROS**，是大厂标准工作流。

### 周计划

| 周次 | 主题 | 每天做什么 | 验证 |
|:---:|:----|:---------|:----|
| 26 | ROS 基础概念 | 3天学 ROS2 核心概念（Node/Topic/Service/Action）；2天在 Ubuntu 里跑通 turtlesim 和 rqt | 能用命令行控制 turtlesim |
| 27 | ROS Toolbox 入门 | 3天在 Simulink 中用 ROS Subscribe/Publish 模块；2天搭"ROS话题 → Simulink PID → ROS话题"闭环 | Simulink 能收发 ROS 消息 |
| 28 | **Simulink 模型 → ROS 节点** | 3天把 Phase 1 的 PID 模型用 ROS Toolbox 生成 ROS 节点；2天部署到 Ubuntu 环境运行 | 生成的节点能独立运行 |
| 29 | Gazebo 仿真集成 | 3天在 Gazebo 中加载 TurtleBot3；2天用 Simulink 模型控制 Gazebo 中的机器人 | 仿真中机器人按指令运动 |
| 30 | **实战：位置跟踪** | 5天搭完整系统：目标位置 → Simulink PID → ROS cmd_vel → Gazebo 机器人 → 位置反馈 → Simulink | Gazebo 中机器人跟踪目标轨迹 |
| 31 | 缓冲 + 整理 | 整理代码，写 README，提交 GitHub | 一个完整的 Simulink+ROS Demo |

### 简历补充
```
掌握 Simulink ROS Toolbox，可将控制模型自动生成 ROS 节点并部署到机器人
具备 Simulink + ROS + Gazebo 联合仿真经验
```

## 答疑

### Q：中间断了怎么办？
Phase 1-2 有弹性，Phase 3 是核心不能断太久。如果 Phase 3 断了超过2周，回去复习再继续。

### Q：没有 dSPACE 硬件怎么办？
走 Phase 4 路径B（Simulink Desktop Real-Time + 概念学习），重点保证 Phase 3 代码生成的质量。面试时说"熟悉 dSPACE 工作流但缺硬件实操"——比完全不懂强很多。

### Q：ROS 要学到多深？
不需要成为 ROS 专家。你的定位是"用 Simulink 生成 ROS 节点"而不是"手写 ROS 节点"。Phase 5 的6周足够打通这个工作流。

### Q：每天时间不够30分钟怎么办？
即使每天只有15分钟，也比一周攒3小时集中学好——Simulink 是肌肉记忆型技能，频率比时长重要。
